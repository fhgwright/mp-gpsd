We want to extend this table. To report on a GPS, please tell us the following:

  1. The GPS model name
  2. Whether or not it works
  3. If possible, an URL to a technical reference on it
  4. The GPS and (if applicable) USB-to-serial chip it uses. (Look at the output of lsusb(1), it may identifty the USB-to-serial chip.)
  5. The interface type (RS232, USB, CF)
  6. The vendor
  7. The gpsd driver type (almost always 'n')
  8. Version of gpsd you tested with
  9. NMEA version it emits, or '?' if the documentation doesn't say
  10. A list of NMEA sentences it emits
  11. Notes on the device

You should be able to determine many of these things by running gpsprobe against the device and looking at the output.

Notes on chipset support:

SiRF-II: A technical manual describing NMEA extensions supported by this chipset is available from Globalsat's U.S. reseller. For SiRF binary ' protocol itself, look here.

A technical spec for the Evermore chipset is available.

Zodiac: See the Zodiac Serial Data Interface Specification

PL2303: support is broken in late 2.4 Linux kernels (it broke after 2.4.18) but good in 2.6.8 and later.

RS232 levels:

TTL level RS-232 uses 0v and +5v, ordinary RS-232 uses -12v to -5v and +5v to +12 volts for the signal (IIRC). These are not compatible, and attempting to combine them can zorch your GPS. Converters do exist, see Google.
